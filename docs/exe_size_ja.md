# exe ファイルのサイズについて

## exe ファイルが約150MBと大きい理由

このアプリの exe ファイルが大きいのは、
**.NET の自己完結型（self-contained）アプリ**としてビルドされているためです。

---

## 自己完結型（self-contained）とは？

このアプリには **.NET ランタイム本体が exe に含まれています**。

そのため：

- .NET を別途インストールする必要がありません
- 追加の DLL や依存ライブラリが不要です
- ダウンロード後すぐに実行できます
- .NET のバージョン違いによるトラブルが起きません

---

## なぜこの方式を採用したのか

### ユーザー側のメリット
- インストール不要で、すぐに使える
- 実行環境によるトラブルが起きにくい
- オフラインでも動作する
- 技術に詳しくない方でも安心して利用できる

### 配布・運用上のメリット
- 動作の再現性が高い
- 環境依存の不具合を最小限にできる
- 単一 exe ファイルとして配布できる

---

## サイズは正常？

はい、**想定どおりで正常です**。

Windows 用の自己完結型 .NET アプリでは、
100〜200MB 程度になることは一般的です。

---

## サイズを小さくすることはできる？

技術的には可能ですが、以下のデメリットがあります：

- .NET の事前インストールが必要になる
- 環境によって起動しない可能性がある
- 多言語や UI 周りで不具合が出やすくなる

本プロジェクトでは、
**「確実に動くこと」「簡単に使えること」**を優先しています。
